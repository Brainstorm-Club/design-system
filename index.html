02

Tipografia

La macchina da scrivere del logo porta tutta la personalità. Il display è monospace, sempre minuscolo come nel marchio. Il corpo del testo è un sans umanista ad alta leggibilità — nei giochi si legge tanto.

Display · brand

brainstorm

Courier Prime → Courier New. Minuscolo, spaziatura stretta. Titoli, marchio, bottoni.

Body · testo

Atkinson Hyperlegible → system-ui. Corpo, interfaccia, testi lunghi.

06

Voce & tono

Diretti, caldi, mai seriosi. Si parla come attorno a un tavolo: seconda persona, frasi corte, il gergo del gioco quando serve.

✓ Sì

  • «Porta un gioco o scegline uno dallo scaffale.»
  • «Martedì si tira: tavoli aperti dalle 21.»
  • Minuscolo per il marchio, sempre.
  • Rosso solo sull'azione principale.

✕ No

  • «L'associazione è lieta di comunicare che…»
  • MAIUSCOLE gridate o toni istituzionali.
  • Rosso ovunque: perde il suo peso.
  • Font decorativi che rubano la scena al typewriter.

08

Sviluppo

Come usare il design system nelle app del club (character builder, Harp Forge, tracker). Un import solo, gli stessi token in CSS e in JavaScript, e i componenti pensati per le schede. Ogni riquadro si copia.

Importare il sistema

Un unico file tira dentro font, token e componenti. Via GitHub Pages funziona come una CDN, senza installare nulla.

<!-- CDN via GitHub Pages -->
<link rel="stylesheet"
      href="https://brainstorm-club.github.io/design-system/brainstorm.css">
<link rel="icon" type="image/png"
      href="https://brainstorm-club.github.io/design-system/assets/favicon.png">
# In un progetto Vite/Vue/TS (come submodule o pacchetto)
import '@brainstorm/design-system/brainstorm.css'

Token in JavaScript / TypeScript

Per i colori dei grafici, il canvas o gli stili dinamici, dove il CSS non arriva. Per i valori che seguono il tema usa cssVar().

import { bsc, cssVar, setTheme } from '@brainstorm/design-system/tokens.js'

ctx.fillStyle = bsc.color.rosso          // '#C2332B'
const serie = bsc.chart                  // palette per i grafici
el.style.color = cssVar('--bsc-primary') // valore risolto col tema attivo
setTheme('light')                        // 'dark' | 'light'

App-bar, hamburger & switch lingua

Header responsivo: nav orizzontale su desktop, hamburger su mobile; menu e switch lingua come dropdown accessibili. Comportamento in ui.js (delega, funziona anche in SPA). Prova qui sotto — hamburger e lingua sono vivi:

Clicca l'hamburger e lo switch lingua (IT/EN).

<header class="bsc-appbar">
  <div class="bsc-appbar__inner">
    <a class="bsc-wordmark" href="/">…</a>

    <!-- switch lingua (dropdown) -->
    <div class="bsc-dropdown bsc-langswitch" data-bsc-dropdown>
      <button class="bsc-dropdown__trigger" data-bsc-dropdown-trigger
              aria-haspopup="listbox" aria-expanded="false">
        <span data-bsc-lang-current>IT</span> ▾
      </button>
      <div class="bsc-dropdown__menu" role="listbox">
        <button class="bsc-dropdown__item" data-bsc-lang="it">IT Italiano</button>
        <button class="bsc-dropdown__item" data-bsc-lang="en">EN English</button>
      </div>
    </div>

    <!-- hamburger (mobile) -->
    <button class="bsc-hamburger" data-bsc-nav-toggle
            aria-controls="main-nav" aria-expanded="false" aria-label="Menu">

<script type="module">
  import { initUI } from '@brainstorm/design-system/ui.js'
  initUI()  // tema + nav + dropdown + lingua
  // l'i18n resta dell'app:
  document.addEventListener('bsc:langchange', e => { i18n.locale = e.detail.lang })
</script>

Tema: uno switch per tutte le app

Un componente e un modulo condivisi, così lo switch carbone ⇄ carta è identico ovunque: default scuro, la scelta si ricorda (localStorage, chiave bsc-theme). Il toggle qui nella barra usa esattamente questo.

provalo: cambia il tema e ricarica — se lo ricorda, funziona. 
<!-- 1. anti-flash: nell'<head>, PRIMA del CSS -->
<script>try{document.documentElement.setAttribute('data-theme',
  localStorage.getItem('bsc-theme')||'dark')}catch(e){}</script>

<!-- 2. il pulsante (uno o più, ovunque) -->
<button class="bsc-theme-toggle" data-bsc-theme-toggle>◐</button>

<!-- 3. avvia una volta all'inizio -->
import { initTheme } from '@brainstorm/design-system/theme.js'
initTheme()

// In Vue/React gestisci il tuo pulsante con:
import { getTheme, setTheme, toggleTheme } from '@brainstorm/design-system/theme.js'
// e ascolta l'evento document 'bsc:themechange'
